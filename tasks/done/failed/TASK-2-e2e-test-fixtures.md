# TASK-2: Create E2E test fixtures and fixture helper

## Goal
`e2e/fixtures/` contains one valid test file per media category (image, video, audio, data, archive), a `scripts/gen-fixtures.sh` generates the binary media files via ffmpeg, and `e2e/helpers/fixtures.ts` exports absolute paths to all fixtures.

## Context
The Fade E2E test suite (being built across multiple tasks) needs small, known-good fixture files to drive conversion and UI tests. These fixtures must be real files that the Tauri backend can read — mocks won't work here because `convert_file` is invoked via IPC and hits the actual Rust backend.

**Why each fixture type:**
- `test.png` — image conversions (PNG → WebP, JPEG, etc.)
- `test.mp4` — video conversions (MP4 re-encode, format change)
- `test.wav` — audio conversions (WAV → MP3, FLAC, etc.)
- `test.csv` — data conversions (CSV → JSON, TSV, etc.)
- `test.json` — data conversions (JSON → CSV, YAML, etc.)
- `test.zip` — archive conversions (ZIP → TAR.GZ, etc.)

**Generation approach:** Binary media files (PNG, MP4, WAV) are generated by `scripts/gen-fixtures.sh` using ffmpeg (which must be in PATH — it is, per CLAUDE.md). Text fixtures (CSV, JSON) are written as literal files. The zip is created by the script. All generated files are written to `e2e/fixtures/`.

**Fixture helper:** `e2e/helpers/fixtures.ts` exports `FIXTURES` as a const object with `png`, `mp4`, `wav`, `csv`, `json`, `zip` keys mapping to absolute resolved paths. This gives tests a single import point that works regardless of CWD.

**CI note:** `scripts/gen-fixtures.sh` must be run before `npm run test:e2e`. Add a step to `.github/workflows/ci.yml` to run it: `bash scripts/gen-fixtures.sh`. This step goes before the E2E smoke test step added in TASK-1.

## In scope
- `e2e/fixtures/` directory (new) — all fixture files
- `e2e/helpers/fixtures.ts` (new) — path helper
- `scripts/gen-fixtures.sh` (new) — media generation script
- `.github/workflows/ci.yml` — add one new step to run gen-fixtures.sh before e2e tests
- `.gitignore` — add `e2e/fixtures/*.mp4`, `e2e/fixtures/*.wav`, `e2e/fixtures/*.png` to avoid committing binary blobs; commit only `test.csv`, `test.json`, `test.zip`

## Steps

1. **Create `e2e/fixtures/` directory** if it doesn't exist.

2. **Create `e2e/fixtures/test.csv`** with literal content:
   ```
   name,age,city
   Alice,30,New York
   Bob,25,London
   Carol,35,Tokyo
   David,28,Sydney
   ```

3. **Create `e2e/fixtures/test.json`** with literal content:
   ```json
   {
     "title": "Fade test fixture",
     "version": 1,
     "tags": ["test", "e2e"],
     "meta": { "created": "2026-01-01", "size": "small" }
   }
   ```

4. **Create `scripts/gen-fixtures.sh`** (executable: `chmod +x`). The script must:
   - Set `set -euo pipefail`
   - Define `OUT="$(dirname "$0")/../e2e/fixtures"` and `mkdir -p "$OUT"`
   - Generate `test.png`: `ffmpeg -y -f lavfi -i color=c=0x4A90E2:size=320x240:rate=1 -frames:v 1 "$OUT/test.png"`
   - Generate `test.mp4`: `ffmpeg -y -f lavfi -i color=c=0x4A90E2:size=320x240:rate=30 -t 3 -c:v libx264 -pix_fmt yuv420p "$OUT/test.mp4"`
   - Generate `test.wav`: `ffmpeg -y -f lavfi -i sine=frequency=440:duration=2 -ar 44100 "$OUT/test.wav"`
   - Create `test.zip`: write a temp text file `echo "Fade test fixture" > /tmp/fade_fixture.txt` then `cd /tmp && zip -j "$OLDPWD/$OUT/test.zip" fade_fixture.txt` then `rm /tmp/fade_fixture.txt`
   - Print "Fixtures generated in $OUT" when done

5. **Run `scripts/gen-fixtures.sh` locally** to verify all files are created. Confirm `test.png`, `test.mp4`, `test.wav`, `test.zip` exist in `e2e/fixtures/`.

6. **Create `e2e/helpers/fixtures.ts`**:
   ```ts
   import { resolve } from 'path'

   const dir = resolve(__dirname, '../fixtures')

   export const FIXTURES = {
     png:  resolve(dir, 'test.png'),
     mp4:  resolve(dir, 'test.mp4'),
     wav:  resolve(dir, 'test.wav'),
     csv:  resolve(dir, 'test.csv'),
     json: resolve(dir, 'test.json'),
     zip:  resolve(dir, 'test.zip'),
   } as const
   ```

7. **Update `.gitignore`** — add these lines to keep binary media out of git:
   ```
   e2e/fixtures/*.png
   e2e/fixtures/*.mp4
   e2e/fixtures/*.wav
   ```
   The `.zip`, `.csv`, and `.json` fixtures ARE small enough to commit.

8. **Update `.github/workflows/ci.yml`** — add a step before the `E2E smoke test` step:
   ```yaml
   - name: Generate E2E fixtures
     run: bash scripts/gen-fixtures.sh
   ```
   This step goes after the `Build app (automation feature)` step and before `E2E smoke test`.

## Success signal
- `e2e/fixtures/test.csv`, `test.json`, `test.zip` exist and are non-empty.
- Running `bash scripts/gen-fixtures.sh` from the repo root creates `test.png`, `test.mp4`, `test.wav` in `e2e/fixtures/`.
- `e2e/helpers/fixtures.ts` exists and the `FIXTURES` object exports six keys.
- `.gitignore` contains the three binary fixture patterns.
- `.github/workflows/ci.yml` has the `Generate E2E fixtures` step before the E2E test step.

## Notes
- ffmpeg must be in PATH (`brew install ffmpeg` on macOS dev). On macOS-14 GitHub Actions runners, ffmpeg is available via `brew install ffmpeg` or may already be present — add a `brew install ffmpeg` step to CI before the fixtures step if `ffmpeg` is not available by default.
- `__dirname` in `e2e/helpers/fixtures.ts` works when wdio runs specs via ts-node/tsx. If it fails, use `import.meta.url` and `fileURLToPath` instead for ESM compatibility.
- The ZIP fixture uses `zip -j` to flatten the directory (no path prefix inside the zip). This matches what Fade expects for archive inputs.
